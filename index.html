2. ‚òÅÔ∏è Cloud Architecture

The backend leverages a serverless-first approach on AWS to ensure scalability, cost-efficiency, and high availability. AWS API Gateway serves as the single entry point for all client requests, routing them to the appropriate backend services.

  • AWS API Gateway: Manages all API endpoints, request routing, and authentication.
  • AWS Lambda: Executes core business logic for various services (Product, Cart, Order, Customer, Homepage).
  • AWS RDS (MySQL): Provides a managed relational database for persistent data storage.
  • AWS S3: Securely stores product images and other static assets.
  • AWS IAM: Manages fine-grained access control and permissions for secure resource access.
  • AWS CloudWatch: Centralized logging and monitoring for all services.
  • Architecture Focus: Microservices pattern, serverless computing, event-driven design.

5. üìä Monitoring & Observability

Effective logging and monitoring are crucial for maintaining the health and performance of the backend services. AWS CloudWatch is utilized to collect logs, metrics, and set up alarms, providing real-time insights into application behavior and operational issues.

  • CloudWatch Logs: Centralized collection of all application logs from FastAPI and other services for easy access and analysis.
  • Custom Metrics: Key performance indicators (KPIs) and business metrics are captured and sent to CloudWatch Metrics for granular tracking.
  • Alarms and Notifications: Configured CloudWatch Alarms trigger notifications (e.g., via SNS) for critical events or performance thresholds, enabling proactive issue resolution.
  • CloudWatch Dashboards: Custom dashboards provide a consolidated, real-time view of application health, performance, and operational status.
